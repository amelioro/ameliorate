# Ameliorate Blog

Built with [Nextra](https://nextra.site) using `nextra-theme-docs` so the layout and styling stay consistent with [docs-site/](../docs-site/). Somewhat vibe-coded, seems ok to not be perfect for now.

## Note on framework choice

Nextra's docs-theme isn't really designed for a blog (we opted out of `nextra-theme-blog` because its styling felt dated, and we wanted consistent styling with the docs). Plain Next.js + MDX would be a closer fit for just the blog, but if we're going to spend time migrating, the better target is moving both [docs-site/](../docs-site/) and this site to Astro — it handles both docs and blogs well. For now, sticking with Nextra is the lowest-effort path; revisit when the styling friction starts to outweigh the migration cost.

## Setup

```sh
npm install
npm run dev # http://localhost:3009/blog
```

## Adding a New Post

1. Create a directory under `app/posts/` named `<YYYY-MM-DD>-<primary-tag>-<brief-title>/` (e.g. `2026-04-01-update-new-feature/`).
2. Inside it, add a `page.mdx` with frontmatter + the `<PostMeta />` component:

   ```mdx
   ---
   title: "Your Post Title"
   author: "Your name"
   date: YYYY-MM-DDT12:00
   description: "Short description for cards and RSS."
   tags:
     - update
   ---

   import { PostMeta } from "../../components/PostMeta";

   <PostMeta />

   Your content here. Use `##` and `###` headings — the right-side TOC pane picks them up automatically.
   ```

3. Add the post to `app/posts/_meta.ts` (at the top — newest first):

   ```ts
   "2026-04-01-update-new-feature": "4/1/26: Brief Title",
   ```

4. Run `npm run dev` and verify:
   - Post appears on the All Posts page and in the sidebar.
   - The `<PostMeta />` block renders title, date, and author.
   - The right-side TOC pane lists your `##` / `###` headings.
5. Push to `main` — Netlify auto-deploys the `blog-site/` site.

## Tags

Tags are recorded in frontmatter (and used by the RSS feed in the future) but are intentionally not displayed yet — the only tag in use right now is `update`, so the UI would just be noise. When a second tag type appears, re-introduce a tag list on the All Posts page, add a `/tags/<tag>` endpoint to view posts with a tag, and add a `<PostMeta />` tag row.

## Cross-posting to Substack

Once a post is live on `ameliorate.app/blog`:

1. Copy the rendered content from into Substack's editor.
2. At the very top of the Substack version, add a line linking back to the original, e.g.:
   _See [the original version of this post on ameliorate.app/blog](https://ameliorate.app/blog/posts/<your-post-slug>). It may have better formatting._
3. Publish.

Substack unfortunately does not allow setting a canonical URL to an external site.

### Notes on decision to use ameliorate.app/blog + Substack

Considered three options for where to host the blog:

1. just Substack. didn't go with this because a. a solid blog post directly on `ameliorate.app` could be good for SEO, and b. we might want more control over the blog formatting e.g. to add a Tabs component to easily compare different info structures.
2. `ameliorate.app/blog` + Buttondown. this could be nice because people can subscribe via it but it also actually let's you set a canonical URL. also it has a great (but paid $9/mo) feature to just send emails based on RSS feed. didn't go with it because a. didn't want to deal with having to switch subscribers over to Buttondown - can always do that later.
3. `ameliorate.app/blog` + Substack. went with this because it allows the SEO benefits with the least effort. also it's nice on `ameliorate.app` to not have to clarify "what is Ameliorate?" in every post, because people can just go to the home page.

Might be worth switching to `ameliorate.app/blog` + Buttondown to fully optimize SEO but not really worth the effort right now. Note: might also want to consider if it's useful to keep Substack for the commenting / "like" functionality. Not sure if that's necessary.
